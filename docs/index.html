Release Workflow

The site is static, but release metadata is generated during deploy. GitHub Releases owns the version and DMG URL.

Step 1

Ship a GitHub release

Upload a new PhoneRelay.dmg asset to a public release such as v1.0.4.

Step 2

Pages writes metadata

The deploy workflow reads the latest release with GITHUB_TOKEN and writes release.json.

Step 3

Download link updates

The button, release link, version label, file size, and published date update from the metadata file.

Requirements

Phone Relay is built for local Android mirroring on macOS. The release should stay signed and notarized before publishing.

Mac

macOS 13 or newer

The app uses native SwiftUI and macOS networking, notification, and USB permissions.

Android

USB or Wireless debugging

Connect by USB first, or pair over Wi-Fi with Android's Wireless debugging QR code.

Trust

Signed and notarized DMG

Published builds should be Developer ID signed and stapled before the release asset is replaced.
